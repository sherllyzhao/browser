# 多账号登录信息存取指南

## 概述

本文档详细说明如何在多账号模式下存取和管理第三方平台的登录信息（Cookies、localStorage、sessionStorage、IndexedDB）。

## 数据存储架构

### Session 分区设计

```
主窗口 BrowserView
└── session: persist:browserview （独立，仅用于浏览）

平台账号（每个账号独立 session）
├── 抖音
│   ├── 账号A → session: persist:douyin_dy_xxx_1
│   └── 账号B → session: persist:douyin_dy_xxx_2
├── 小红书
│   └── 账号X → session: persist:xiaohongshu_xhs_xxx_3
├── 百家号
├── 微信公众号
└── 视频号
```

### 物理存储位置

```
%APPDATA%\运营助手\                          # 根目录（或便携版的 %LOCALAPPDATA%\运营助手-Portable\）
├── global-storage.json                      # 自己平台登录信息 + 账号元数据
└── Partitions\
    ├── browserview\                         # 主窗口持久化 Session
    │   └── Network\Cookies                  # 第三方平台 Cookie
    ├── douyin_dy_xxx_1\                     # 抖音账号A Session
    │   └── Network\Cookies
    ├── douyin_dy_xxx_2\                     # 抖音账号B Session
    │   └── Network\Cookies
    └── xiaohongshu_xhs_xxx_3\               # 小红书账号 Session
        └── Network\Cookies
```

## API 列表

### 1. 账号管理 API

#### 获取账号列表

```javascript
// 获取指定平台的所有账号
const accounts = await window.browserAPI.getAccounts('douyin');
// 返回: [
//   { id, nickname, avatar, platformUid, createdAt, lastUsedAt },
//   ...
// ]

// 获取所有平台的所有账号
const allAccounts = await window.browserAPI.getAllAccounts();
// 返回: { douyin: [...], xiaohongshu: [...], ... }
```

#### 添加账号

```javascript
// 授权成功后添加账号
const result = await window.browserAPI.addAccount('douyin', {
  nickname: '账号昵称',
  avatar: '头像URL',
  platformUid: '平台用户ID'  // 用于去重
});

// 返回值说明：
// {
//   success: true,
//   accountId: 'douyin_xxx_1',
//   isNew: true              // true: 新添加，false: 更新现有账号
// }
```

#### 删除账号

```javascript
// 删除账号（同时清理对应的 session 数据）
const result = await window.browserAPI.removeAccount('douyin', 'douyin_xxx_1');
// 返回: { success: true }
```

#### 更新账号信息

```javascript
const result = await window.browserAPI.updateAccount('douyin', 'douyin_xxx_1', {
  nickname: '新昵称',
  avatar: '新头像URL'
});
// 返回: { success: true, account: {...} }
```

#### 检查账号是否存在

```javascript
// 通过平台用户ID检查账号是否已存在
const result = await window.browserAPI.accountExists('douyin', '平台用户ID');
// 返回:
// { exists: true, accountId: 'douyin_xxx_1', account: {...} }
// 或
// { exists: false }
```

#### 获取单个账号信息

```javascript
const result = await window.browserAPI.getAccount('douyin', 'douyin_xxx_1');
// 返回: { success: true, account: { id, nickname, avatar, ... } }
```

#### 获取当前窗口账号信息

```javascript
// 在发布脚本中获取当前窗口对应的账号信息
const result = await window.browserAPI.getCurrentAccount();
// 返回: {
//   success: true,
//   platform: 'douyin',
//   accountId: 'douyin_xxx_1',
//   account: { id, nickname, avatar, ... }
// }
```

### 2. 会话数据存取 API

#### 获取完整会话数据

```javascript
// 获取完整会话数据（Cookies + localStorage + sessionStorage + IndexedDB）
// 用于授权后将完整登录状态存储到后台
const result = await window.browserAPI.getFullSessionData('douyin.com');

if (result.success) {
  const sessionData = result.data;
  // sessionData 包含 cookies, localStorage, sessionStorage, indexedDB
  // 可以提交到后台存储
}
```

#### 恢复账号会话数据

```javascript
// 从后台获取账号的完整会话数据后，恢复到账号 session
const result = await window.browserAPI.restoreAccountSession(
  'douyin',           // platform
  'douyin_xxx_1',     // accountId
  sessionData         // 会话数据对象或 JSON 字符串
);

// 返回值说明：
// {
//   success: true,
//   results: {
//     cookies: { restored: 15, failed: 0 },
//     localStorage: { restored: 3, failed: 0 },
//     sessionStorage: { restored: 1, failed: 0 },
//     indexedDB: { restored: 2, failed: 0 }
//   }
// }
```

#### 清空账号的所有 Cookies

```javascript
// 清空指定账号的所有 cookies
const result = await window.browserAPI.clearAccountCookies('douyin', 'douyin_xxx_1');
// 返回: { success: true, deletedCount: 10 }
```

#### 迁移到新账号（授权窗口使用）

```javascript
// 授权成功后，将临时 session 的 cookies 迁移到新账号
const result = await window.browserAPI.migrateToNewAccount('douyin', {
  nickname: '获取到的昵称',
  avatar: '获取到的头像',
  platformUid: '获取到的平台用户ID'
});

// 返回值说明：
// {
//   success: true,
//   accountId: 'douyin_xxx_1',  // 新创建的账号ID
//   isNew: true,                // 是否为新账号
//   migratedCount: 20           // 迁移的 cookie 数量
// }
```

### 4. 登录有效期与提醒机制

- Cookie 会持久化 `expirationDate` 字段，并随会话数据一起存储/恢复。
- 恢复会话时，如果某个 Cookie 没有 `expirationDate`，系统会使用“当前时间 + 1 年”的兜底过期时间。
- 登录有效性判断不只看 Cookie，还会结合业务 token 过期时间（`login_expires`）和接口鉴权结果（例如 `401`）。
- 检测到登录已过期时，应用会弹出提示并跳转到登录页。
- 应用会在 token 到期前默认 10 分钟提前提示一次（每分钟检查一次；同一过期时间只提示一次，避免刷屏）。

## 使用流程

### 授权流程

```javascript
// 1. 打开授权窗口（临时 session，不保存到本地）
const authResult = await window.browserAPI.openNewWindow(authUrl, {
  useTemporarySession: true
});

// 2. 在授权页脚本中，检测登录成功后获取用户信息
async function onAuthSuccess(userInfo) {
  // 迁移 cookies 到新账号（自动创建或更新）
  const result = await window.browserAPI.migrateToNewAccount('douyin', {
    nickname: userInfo.nickname,
    avatar: userInfo.avatar,
    platformUid: userInfo.uid
  });

  if (result.success) {
    console.log('新账号ID:', result.accountId);

    // 获取完整会话数据（用于提交到后台）
    const sessionResult = await window.browserAPI.getFullSessionData('douyin.com');
    if (sessionResult.success) {
      // 提交完整会话数据到后台保存
      await fetch('https://api.example.com/save-session', {
        method: 'POST',
        body: JSON.stringify({
          accountId: result.accountId,
          sessionData: sessionResult.data
        })
      });
    }

    // 通知首页刷新账号列表
    window.browserAPI.sendToHome({
      type: 'account-added',
      platform: 'douyin',
      accountId: result.accountId
    });

    // 关闭授权窗口
    window.browserAPI.closeCurrentWindow();
  }
}
```

### 发布流程（推荐方式）

```javascript
// 1. 首页选择要发布的账号并打开发布窗口
async function publishToAccount(platform, accountId, publishUrl) {
  // 从后台获取该账号的会话数据
  const response = await fetch(
    `https://api.example.com/get-session/${accountId}`
  );
  const { sessionData } = await response.json();

  // 打开发布窗口，传入 sessionData 参数
  // 浏览器会自动清空旧 cookies 并恢复新的会话数据
  const result = await window.browserAPI.openNewWindow(publishUrl, {
    platform: platform,
    accountId: accountId,
    sessionData: sessionData  // 浏览器会自动清空并恢复
  });

  if (result.success) {
    // 存储发布数据
    await window.browserAPI.setGlobalData(
      `publish_data_window_${result.windowId}`,
      {
        title: '视频标题',
        content: '视频描述'
      }
    );
  }
}

// 2. 在发布脚本中获取账号信息和发布数据
async function onPublishPageLoaded() {
  // 获取当前窗口的账号信息
  const accountInfo = await window.browserAPI.getCurrentAccount();
  console.log('当前账号:', accountInfo.account?.nickname);

  // 获取发布数据
  const windowId = await window.browserAPI.getWindowId();
  const publishData = await window.browserAPI.getGlobalData(
    `publish_data_window_${windowId}`
  );

  // 执行发布逻辑...
}

// 3. 发布完成后清理
async function onPublishComplete() {
  const accountInfo = await window.browserAPI.getCurrentAccount();
  if (accountInfo.success) {
    // 清空登录状态（避免登录信息累积）
    await window.browserAPI.clearAccountCookies(
      accountInfo.platform,
      accountInfo.accountId
    );
    // 关闭窗口
    await window.browserAPI.closeCurrentWindow();
  }
}
```

### 发布流程（手动恢复方式）

```javascript
// 如果需要更精细的控制，可以手动恢复会话
async function publishWithManualRestore(platform, accountId, publishUrl) {
  // 1. 从后台获取会话数据
  const response = await fetch(
    `https://api.example.com/get-session/${accountId}`
  );
  const { sessionData } = await response.json();

  // 2. 清空该账号的旧 cookies（可选，如果 openNewWindow 传了 sessionData 则自动清空）
  // await window.browserAPI.clearAccountCookies(platform, accountId);

  // 3. 恢复会话数据
  const restoreResult = await window.browserAPI.restoreAccountSession(
    platform,
    accountId,
    sessionData
  );
  if (restoreResult.success) {
    console.log(`已恢复 ${restoreResult.results.cookies.restored} 个 cookies`);
  }

  // 4. 打开发布窗口
  const windowResult = await window.browserAPI.openNewWindow(publishUrl, {
    platform: platform,
    accountId: accountId
  });

  return windowResult;
}
```

## 常见问题

### Q: 如何区分新账号和更新现有账号？

A: 调用 `addAccount` 或 `migrateToNewAccount` 时，返回值中的 `isNew` 字段可以判断：
- `isNew: true` - 新创建的账号
- `isNew: false` - 更新了现有账号（通过 `platformUid` 去重）

### Q: 能否不清空旧 cookies 直接打开窗口？

A: 可以，但不推荐。这样会导致登录信息累积。建议的做法：
- 如果从后台有最新的 `sessionData`，通过 `openNewWindow` 传入 `sessionData` 参数，让浏览器自动清空并恢复
- 或者手动调用 `clearAccountCookies` 再打开窗口

### Q: IndexedDB 数据如何恢复？

A: `restoreAccountSession` 目前只支持恢复 Cookies。IndexedDB 数据需要：
1. 通过 `getFullSessionData` 获取数据库数据
2. 在窗口打开后通过页面脚本手动恢复（使用 IndexedDB API）

### Q: 如何监听会话数据变化？

A: 使用 `onSessionUpdated` 回调：

```javascript
window.browserAPI.onSessionUpdated((data) => {
  console.log('发布窗口关闭，已自动保存会话数据');
  console.log('平台:', data.platform);
  console.log('账号ID:', data.accountId);
  // data 包含: windowId, platform, accountId, cookies, publishData, timestamp
});
```

### Q: 可以把 Cookie 改成永不过期来保持永久登录吗？

A: 不建议且不可依赖。即使本地 Cookie 过期时间被改长，服务端仍会按 token 生命周期和鉴权状态判定是否登录；token 失效后接口会返回未登录。

### Q: 如果只改本地 Cookie 过期时间，后台能知道吗？

A: 后台通常不会直接知道你改过本地 `expires/expirationDate` 属性，但会通过请求鉴权结果感知 token 是否仍有效。

### Q: 提前过期提醒时间可以改吗？

A: 可以。当前默认提前 10 分钟提醒，配置项在 `renderer.js` 的 `TOKEN_EXPIRY_REMINDER_SECONDS` 常量，单位为秒。

## 总结

- **存储方式**: 每个账号有独立的 Session 分区，元数据存在 `global-storage.json`
- **获取数据**: 使用 `getFullSessionData` 获取完整登录信息（Cookies + 存储数据）
- **恢复数据**: 使用 `restoreAccountSession` 或 `openNewWindow` 的 `sessionData` 参数恢复
- **推荐流程**: 后台存储完整 `sessionData`，发布时通过 `openNewWindow` 传入，浏览器自动完成清空和恢复
- **清理数据**: 使用 `clearAccountCookies` 清理登录状态，避免累积
