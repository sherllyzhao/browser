## 测试步骤

### 1. 重启浏览器
```bash
# 停止当前运行的浏览器，然后重新启动
npm start
```

### 2. 打开首页并查看控制台
1. 在运营助手浏览器中访问首页 `http://localhost:5173/`
2. 按 F12 打开浏览器 DevTools（注意是内容页面的 DevTools，不是控制面板的）
3. 查看 Console 标签，应该看到：
   ```

   ```

### 3. 打开授权页面测试
1. 在首页停留，保持 DevTools 打开
2. 导航到授权页面 `http://localhost:5173/#/short-video/auth`
3. 查看 Console，应该看到：

### 4. 手动发送测试消息
在授权页面的 Console 中执行：

## 如果还是收不到消息

### 检查清单：

1. **确认在首页**
   - 首页 URL 必须以 `http://localhost:5173/` 开头
   - 可以有 hash 路由，如 `http://localhost:5173/#/short-video/account`

2. **确认 browserAPI 可用**
   - 在授权页面控制台输入：`console.log(window.browserAPI)`
   - 应该看到一个对象，包含 `sendToHome` 方法

3. **查看主进程日志**
   - 打开运行 `npm start` 的终端
   - 查看是否有 `[IPC] 收到 content-to-home 消息` 的日志

4. **检查首页监听器**
   - 在首页控制台输入：
     ```javascript
     window.addEventListener('message', (e) => {
       console.log('收到 postMessage:', e.data);
     });
     ```
   - 然后在授权页面再次调用 `window.__sendAuthSuccess__()`
   - 查看首页是否收到消息

5. **检查 AddAccount 组件是否挂载**
   - 在首页控制台应该能看到 `AddAccount组件已挂载`
   - 如果没有，说明组件没有加载，监听器也不会注册

## 调试技巧

### 方法1: 直接在首页测试 postMessage
在首页控制台执行：
```javascript
window.postMessage({ type: 'FROM_OTHER_PAGE', data: { type: 'auth-success' } }, '*');
```
这应该立即触发授权成功提示。

### 方法2: 检查监听器是否注册
在首页控制台执行：
```javascript
// 手动注册监听器
window.browserAPI.onMessageFromOtherPage((message) => {
  console.log('手动监听器收到消息:', message);
});
```

### 方法3: 查看 IPC 通信
在主进程添加日志，查看 IPC 消息是否正确传递。

## 正式使用

测试成功后，你需要根据实际的授权流程修改 auth.js:64-70 的触发条件：

```javascript
// 例如：授权成功后 URL 会带 code 参数
if (urlParams.get('code')) {
  sendAuthSuccessMessage();
}

// 或者：监听特定按钮点击
document.querySelector('.auth-confirm-btn')?.addEventListener('click', () => {
  sendAuthSuccessMessage();
});
```
