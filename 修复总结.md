# 多窗口发布成功率低 - 问题修复总结

## 问题诊断

**症状**：客户反映同时打开5个发布窗口时，只有1-2个能成功，其他需要重新发布。

**根本原因**：**Session Cookie 并发竞态条件**

多个窗口同时操作同一个 `session.cookies` 实例，导致清空和恢复操作相互覆盖：

```javascript
// 问题代码流程（5个窗口并发）
窗口A: 清空100个旧cookies → 恢复50个新cookies → 刷盘
窗口B: 清空100个旧cookies (把A刚恢复的删了) → 恢复50个新cookies → 刷盘
窗口C: 清空100个旧cookies (把B刚恢复的删了) → ...
结果：只有最后刷盘的窗口有正确的 cookies ❌
```

## 修复方案

**核心思路**：添加 Session 级别的操作队列，确保对同一 session 的操作串行执行。

### 代码修改

#### 1. 添加队列机制（main.js:13099-13169）

```javascript
// Session 操作队列
const sessionOperationQueues = new Map();

async function executeInSessionQueue(partitionName, operation, label) {
  // 等待前面所有操作完成
  const previousOperation = queue[queue.length - 1] || Promise.resolve();
  
  // 串行执行
  const currentOperation = previousOperation.then(() => operation());
  
  queue.push(currentOperation);
  return currentOperation;
}
```

#### 2. 包裹 Session 恢复（main.js:9871-10260）

```javascript
if (effectiveSessionData) {
  const partitionName = `persist:${platform}_${accountId}`;
  
  // 🔒 关键修复：放入队列串行执行
  await executeInSessionQueue(partitionName, async () => {
    // 清空旧 cookies
    // 恢复新 cookies
    // 刷盘
  }, `窗口${windowId}-session-restore`);
}
```

## 快速诊断指南

**如果修复后仍有问题**：

1. ❓ **看不到 `[Session Queue]` 日志**
   → 队列未生效，检查代码是否被覆盖

2. ❓ **所有窗口都显示 `[1/1]`**
   → partition 不一致，检查 `partitionName` 计算

3. ❓ **窗口卡在 `⏳ 等待...` 超过30秒**
   → 前序操作死锁，检查是否有未 await 的异步操作

4. ❓ **不同平台的窗口也在排队**
   → partition 计算错误，应该包含 platform 区分
