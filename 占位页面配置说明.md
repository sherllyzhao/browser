# 占位页面配置说明

## 修改背景

### 问题1: GEO tab 选不中
当用户未购买 geo 产品访问 geo 页面时，跳转到 `not-purchase.html?system=geo` 后，GEO tab 没有高亮显示。

### 问题2: 占位页面名称分散
占位页面名称（如 `not-available.html`、`not-auth.html` 等）分散在代码中，难以查找和维护。

---

## 修改内容

### 1. 修复 GEO tab 选不中问题

#### 原因分析

**renderer.js:62 - `getCurrentSystem` 函数缺陷**
```javascript
// 修改前：只检查 not-available.html 和 not-auth.html
if (url.includes('not-available.html') || url.includes('not-auth.html')) {
  const urlObj = new URL(url);
  return urlObj.searchParams.get('system');
}
```
问题：没有检查 `not-purchase.html` 的 `system` 参数。

**main.js:1299, 1362 - geo 权限检查缺陷**
```javascript
// 修改前：跳转后直接 return，没有发送 url-changed 事件
const notPurchaseUrl = `file://${path.join(__dirname, config.placeholderPages.notPurchase)}?system=geo`;
browserView.webContents.loadURL(notPurchaseUrl);
return;  // ❌ 导致 renderer 无法更新 tab 状态
```

#### 修复方案

**renderer.js:62 - 添加 not-purchase.html 检查**
```javascript
// 修改后：检查所有占位页面
const PLACEHOLDER_PAGES = ['not-available.html', 'not-auth.html', 'not-purchase.html', 'login.html'];

function getCurrentSystem(url) {
  if (PLACEHOLDER_PAGES.some(page => url.includes(page))) {
    const urlObj = new URL(url);
    return urlObj.searchParams.get('system');
  }
  // ...
}
```

**main.js:1299, 1362 - 发送 url-changed 事件**
```javascript
// 修改后：跳转前发送事件通知 renderer
const notPurchaseUrl = `file://${path.join(__dirname, config.placeholderPages.notPurchase)}?system=geo`;
mainWindow.webContents.send('url-changed', notPurchaseUrl);  // ✅ 通知 renderer 更新 tab
browserView.webContents.loadURL(notPurchaseUrl);
return;
```

## 使用说明

### 添加新的占位页面

#### 步骤1: 在 config.js 中添加配置
```javascript
placeholderPages: {
  // ... 现有页面
  notAvailable: 'not-available.html',
  notAuth: 'not-auth.html',
  notPurchase: 'not-purchase.html',
  login: 'login.html',
  newPage: 'new-page.html'  // ✅ 新增页面
}
```

#### 步骤2: 在 renderer.js 中添加到数组
```javascript
const PLACEHOLDER_PAGES = [
  'not-available.html',
  'not-auth.html',
  'not-purchase.html',
  'login.html',
  'new-page.html'  // ✅ 新增页面
];
```

#### 步骤3: 在代码中使用配置引用
```javascript
// main.js 中使用
const newPageUrl = `file://${path.join(__dirname, config.placeholderPages.newPage)}?system=${system}`;
browserView.webContents.loadURL(newPageUrl);
```

## 占位页面说明

### 页面类型与用途

| 页面文件 | 配置键名 | 用途 | 触发条件 |
|---------|---------|------|---------|
| `not-available.html` | `notAvailable` | 功能暂未开放 | 系统功能未启用时 |
| `not-auth.html` | `notAuth` | 未登录提示 | 检测到统一登录页时 |
| `not-purchase.html` | `notPurchase` | 未购买产品 | 用户未购买对应产品时 |
| `login.html` | `login` | 本地登录页 | 应用启动时或登出后 |

### URL 参数说明

所有占位页面支持 `system` 参数，用于标识当前系统模块：

```
not-available.html?system=geo    # GEO 系统未开放
not-auth.html?system=aigc        # AIGC 系统未登录
not-purchase.html?system=geo     # GEO 系统未购买
```

**system 参数值**:
- `aigc` - AIGC 系统
- `geo` - GEO 系统
- `browser` - 浏览器系统
- `other` - 其他系统

## 相关文件

- `config.js` - 占位页面配置定义
- `main.js` - 主进程逻辑，权限检查和页面跳转
- `renderer.js` - 渲染进程逻辑，tab 高亮控制
- `not-available.html` - 功能未开放页面
- `not-auth.html` - 未登录页面
- `not-purchase.html` - 未购买产品页面
- `login.html` - 本地登录页面

---

## 注意事项

1. **添加新占位页面时**，必须同时修改 `config.js` 和 `renderer.js` 两个文件
2. **占位页面 URL 参数** `system` 必须与 tab 系统名称一致（aigc/geo/browser/other）
3. **跳转占位页面前** 必须发送 `url-changed` 事件给 renderer，否则 tab 不会高亮
4. **不要直接硬编码页面名称**，始终使用 `config.placeholderPages` 引用
