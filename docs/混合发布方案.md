# 混合发布方案

## 目标

把现有“打开平台发布页 + 注入脚本自动填写/提交”的流程，升级成可审计、可扩展的混合发布流程：

- 官方接口优先：平台有正式授权接口时，由后台任务适配器发布。
- 页面自动化兜底：登录、验证码、手机号认证、平台未开放能力时，打开可见页面处理。
- 托管授权可追踪：记录账号级/批次级授权，不用每次让用户重复确认。
- 私有接口默认禁用：不通过页面传参打开私有接口能力。

## 后续接后台接口的方式

建议新增平台适配器，而不是让注入脚本直接逆向私有接口：

1. 后台返回任务时带 `hybridPublishPolicy`。
2. 主进程打开发布窗口时补齐默认策略。
3. 适配器先判断：
   - 有官方授权接口：后台接口发布。
   - 没有官方接口或遇到风控：打开平台页面自动填表。
4. 所有节点通过 `recordHybridPublishAudit` 写审计。
5. 失败时回退到可见页面，让用户处理登录/验证码/认证。

## 策略边界

- `officialApiOnly: true`：后台接口适配器只允许正式授权接口。
- `allowPrivateApi: false`：页面或后台传入数据不能开启私有接口模式。
- `riskControl.*: show-visible-page`：验证码、登录过期、手机号认证统一回到可见页面。
- 审计会对 `cookie/token/csrf/authorization/password/secret/session` 等敏感字段脱敏。
