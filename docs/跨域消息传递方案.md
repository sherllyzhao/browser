# 跨域消息传递方案

## 问题分析

### ❌ 原方案（失败）
```
首页 (localhost:5173)
  ↓ 存储到 IndexedDB
  ↓ 传递 transferId
抖音页面 (creator.douyin.com)
  ↓ 用 transferId 读取 IndexedDB
  ❌ 失败！跨域无法访问
```

**失败原因**：IndexedDB 是按域名隔离的，不同域名之间无法共享数据。

### ✅ 新方案（成功）
```
首页 (localhost:5173)
  ↓ 直接通过 IPC + postMessage 传递完整数据
抖音页面 (creator.douyin.com)
  ↓ 直接接收完整数据
  ✅ 成功！
```

**优势**：
- 无需 IndexedDB，避免跨域问题
- 数据实时传递，无延迟
- 代码更简洁

## 修改内容

### 1. AddAccount.vue - 发送完整数据
```javascript
// 修改前：发送 transferId，让对方去读 IndexedDB
window?.browserAPI.sendToOtherPage({
  type: 'config-update',
  data: transferId.value
});

// 修改后：直接发送完整授权数据
const authData = {
  company_id: userAndCompanyInfo?.id,
  platform_id: currentPlatform.value?.id,
  platform_value: currentPlatform.value?.value,
  transfer_id: transferId.value,
  timestamp: Date.now(),
};

window?.browserAPI.sendToOtherPage({
  type: 'auth-data',
  data: authData
});
```

### 2. douyin-creator.js - 接收完整数据
```javascript
// 修改前：从 IndexedDB 读取
if (message.type === 'config-update') {
  const authData = await receiveDataFromWindow(message.data);
}

// 修改后：直接接收完整数据
if (message.type === 'auth-data') {
  console.log('[抖音授权] ✅ 收到授权数据:', message.data);
  window.__AUTH_DATA__ = {
    ...window.__AUTH_DATA__,
    ...message.data,
    receivedAt: Date.now()
  };
}
```

### 3. main.js - 广播消息到所有子窗口
```javascript
// 向所有子窗口广播消息
childWindows.forEach((childWindow, index) => {
  if (childWindow && !childWindow.isDestroyed()) {
    childWindow.webContents.executeJavaScript(`
      window.postMessage({ type: 'FROM_HOME', data: ... }, '*');
    `);
  }
});
```

## 常见问题

### Q1: 为什么不用 IndexedDB？
A: IndexedDB 是按域名隔离的。`localhost:5173` 和 `creator.douyin.com` 是不同域名，无法共享 IndexedDB 数据。

### Q2: 数据大小有限制吗？
A: 通过 `postMessage` 传递的数据会被序列化为 JSON。一般的授权数据（几 KB）完全没问题。如果需要传输特别大的数据（MB 级别），需要考虑其他方案。

### Q3: 安全性如何？
A:
- `postMessage` 使用 `'*'` origin 允许所有来源接收，但只在我们自己创建的窗口中
- 数据通过 Electron 的 IPC 通道传递，不经过网络
- 敏感数据（如 token）应该在服务端处理，不要通过前端传递

### Q4: 如果消息丢失怎么办？
A:
- 添加重试机制：抖音页面定时检查 `window.__AUTH_DATA__`，如果没有则重新发送 `PAGE_LOADED`
- 添加超时提示：首页监听 30 秒，如果没收到授权成功消息则提示用户

## IndexedDB 的保留用途

虽然跨域无法使用 IndexedDB，但在**同域**场景下仍然有用：

1. **同域页面间传递大数据**（如 localhost:5173 的不同路由之间）
2. **离线数据缓存**
3. **大文件临时存储**

因此 `common.js` 中的 IndexedDB 相关代码保留，供同域场景使用。
