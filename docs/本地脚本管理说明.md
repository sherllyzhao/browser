# 本地脚本管理系统 - 使用文档

## 功能概述

脚本现在会持久化保存到本地文件系统，应用关闭后不会丢失。所有脚本存储在项目根目录的 `injected-scripts` 文件夹中。

## 文件结构

```
D:\浏览器\运营助手\
├── injected-scripts/           # 脚本存储目录
│   ├── manifest.json          # 脚本清单文件（URL 映射）
│   ├── abc123def456.js        # 脚本文件（MD5 命名）
│   ├── 789ghi012jkl.js
│   └── ...
├── script-manager.js          # 脚本管理模块
└── main.js
```

## 核心功能

### 1. 自动持久化

- **保存脚本**：点击"💾 保存脚本"按钮，脚本会保存到本地文件
- **自动加载**：每次访问对应 URL 时自动从文件加载并注入
- **文件命名**：使用 URL 的 MD5 哈希作为文件名，确保唯一性
- **清单管理**：`manifest.json` 记录 URL 到文件的映射关系

### 2. 脚本列表管理

**在脚本注入面板中：**

- **查看列表**：显示所有已保存的脚本及其大小
- **点击加载**：点击列表中的脚本项可以在编辑器中加载该脚本
- **删除脚本**：点击脚本项右侧的"删除"按钮可删除单个脚本
- **实时更新**：保存或删除脚本后列表自动刷新

### 3. 导入/导出功能

#### 导出脚本

1. 点击"📤 导出"按钮
2. 选择目标文件夹
3. 所有脚本文件和清单会被复制到该文件夹

**导出内容：**
```
导出文件夹/
├── manifest.json              # 清单文件
├── localhost_5173_.js         # 使用友好名称
├── example_com_page_.js
└── ...
```

#### 导入脚本

1. 点击"📥 导入"按钮
2. 选择包含 `manifest.json` 的文件夹
3. 脚本会被导入到本地存储

**用途：**
- 备份和恢复脚本
- 在不同设备间迁移脚本
- 分享脚本配置

### 4. 直接编辑文件

可以在外部编辑器中直接编辑脚本文件：

1. 点击"📁 打开文件夹"按钮
2. 在文件管理器中打开脚本存储目录
3. 使用任何文本编辑器编辑 `.js` 文件
4. 刷新页面即可看到更改

**注意：** 如果修改了文件，请不要删除 `manifest.json` 中的对应记录，否则脚本不会被自动注入。

### 5. 清空所有脚本

点击"🗑 清空全部"按钮可删除所有已保存的脚本。**此操作不可恢复！**

## 使用示例

### 示例 1：保存并自动注入脚本

1. 访问 `http://localhost:5173/`
2. 点击"📝 注入脚本"打开面板
3. 在编辑器中输入：

```javascript
// 自动填充表单
document.getElementById('username').value = 'admin';
document.getElementById('password').value = '123456';

console.log('脚本已自动注入！');
```

4. 点击"💾 保存脚本"
5. 下次访问该页面时脚本会自动注入

### 示例 2：导出和备份脚本

1. 编写了多个页面的注入脚本后
2. 点击"📤 导出"
3. 选择 `D:\backups\my-scripts` 作为导出目录
4. 所有脚本被安全备份

### 示例 3：在外部编辑器中编辑

1. 点击"📁 打开文件夹"
2. 找到对应的脚本文件（可通过 manifest.json 查看 URL 映射）
3. 使用 VS Code 或其他编辑器打开 `.js` 文件
4. 编辑保存
5. 重新访问页面查看效果

### 示例 4：跨设备迁移

**在设备 A：**
1. 点击"📤 导出"，导出到 `U盘/scripts/`

**在设备 B：**
1. 复制 `U盘/scripts/` 到设备
2. 点击"📥 导入"，选择该文件夹
3. 所有脚本立即可用

## 文件格式

### manifest.json 示例

```json
{
  "scripts": {
    "http://localhost:5173/": {
      "filename": "5d41402abc4b2a76b9719d911017c592.js",
      "url": "http://localhost:5173/",
      "savedAt": "2025-01-17T10:30:00.000Z",
      "size": 256
    },
    "https://example.com/page": {
      "filename": "098f6bcd4621d373cade4e832627b4f6.js",
      "url": "https://example.com/page",
      "savedAt": "2025-01-17T11:15:00.000Z",
      "size": 512
    }
  }
}
```

### 脚本文件示例

**文件名：** `5d41402abc4b2a76b9719d911017c592.js`

```javascript
// 这是为 http://localhost:5173/ 注入的脚本
console.log('页面加载完成');

// 修改页面元素
document.querySelector('h1').textContent = '自定义标题';

// 与首页通信
if (window.browserAPI) {
  window.browserAPI.sendToHome({
    type: 'page-ready',
    url: window.location.href
  });
}
```

## 技术细节

### ScriptManager 类 (script-manager.js)

**主要方法：**

- `saveScript(url, script)` - 保存脚本到文件
- `getScript(url)` - 从文件读取脚本
- `getAllScripts()` - 获取所有脚本列表
- `deleteScript(url)` - 删除指定脚本
- `exportScripts(targetDir)` - 导出所有脚本
- `importScripts(sourceDir)` - 从目录导入脚本
- `clearAll()` - 清空所有脚本

### 文件命名规则

使用 MD5 哈希确保文件名唯一且安全：

```javascript
const crypto = require('crypto');
const hash = crypto.createHash('md5').update(url).digest('hex');
const filename = `${hash}.js`;  // 例如：5d41402abc4b2a76b9719d911017c592.js
```

### 数据流程

```
保存脚本:
用户输入 → renderer.js → preload.js → main.js → ScriptManager → 文件系统

加载脚本:
页面加载 → main.js → ScriptManager → 读取文件 → 注入到页面
```

## 注意事项

1. **不要手动删除 manifest.json**，否则所有脚本映射会丢失
2. **备份重要脚本**：使用导出功能定期备份
3. **文件权限**：确保应用有读写 `injected-scripts` 目录的权限
4. **脚本安全**：注入的脚本会在页面上下文中运行，请确保代码安全
5. **清单同步**：如果直接添加 `.js` 文件到文件夹，需要手动更新 manifest.json

## 故障排除

### 脚本没有自动注入

1. 检查脚本是否保存成功（查看列表）
2. 检查 URL 是否完全匹配（包括协议、端口）
3. 打开 DevTools 查看控制台错误
4. 检查 `injected-scripts` 目录是否存在且可访问

### 导入失败

1. 确保目标文件夹包含 `manifest.json`
2. 检查文件格式是否正确
3. 确保文件权限正常

### 脚本列表不更新

1. 重新打开脚本面板
2. 刷新页面
3. 重启应用

## 高级用法

### 批量编辑脚本

使用外部工具批量修改脚本：

```bash
# 在所有脚本中添加相同的代码
cd injected-scripts
for file in *.js; do
  echo "console.log('Added by batch');" >> "$file"
done
```

### 版本控制

可以将 `injected-scripts` 目录加入 Git 进行版本控制：

```bash
git init
git add injected-scripts/
git commit -m "Save injection scripts"
```
