# 登录态保存重构进度

> 更新日期：2026-05-20
> 涉及问题：腾讯/搜狐发布窗口关闭后登录状态保存失败，下次打开仍跳登录页

## 一、问题背景

### 用户感知
- 第一次授权后打开发布窗口正常
- 发布窗口里如果跳过一次登录页，**关浏览器再开发布页就会掉登录**
- 后台收到的 cookies 数据丢失或不完整

### 主要受影响平台
- 腾讯号 (tengxunhao)
- 搜狐号 (sohuhao)
- 其它平台需要逐个验证

---

## 二、根因分析

### 根因 1：sessionData 无条件覆盖本地 session
**位置：** `main.js:7104-7124`

每次打开发布窗口若提供了 `sessionData`，主进程会**先清空账号 session 全部 cookies**，再恢复 `sessionData`。导致用户在窗口里手动登录后写入本地 session 的新 cookies，下次开窗口被旧的 sessionData 覆盖丢失。

### 根因 2：主进程保存接口 body 格式与后台不匹配
**位置：** `main.js:9261-9263`

主进程 `uploadSessionToBackend` 上传格式：
```json
{ "id": "428", "cookies": "..." }
```

但后台 `/api/mediaauth/{tx|sh}info` 接口期望的是与授权脚本 (creator.js) 一致的完整格式：
```json
{ "data": "{nickname, avatar, uid, follow, follower_count, video, company_id, auth_type, cookies, ...}" }
```

主进程的「轻量更新」格式被后台直接拒绝，返回 `{"code":400,"message":"授权失败"}`。

### 根因 3：账号身份字段配置不准
**位置：** `domain-config.js:278-290`

`platformLoginCookies` 配置中：
- 搜狐 `['SUV', 'IPLOC', 'sct']` —— SUV/IPLOC 是访客 cookie（未登录也有）
- 腾讯 `['pgv_pvid', 'RK', 'ptcz']` —— 都是埋点 ID

导致「本地是否已登录」的判断永远为真，账号切换时新 sessionData 进不来。

---

## 三、修复方案

### 方案总览
1. **本地优先 + 账号身份匹配**：本地有真登录态且账号身份匹配时，跳过 sessionData 覆盖
2. **脚本侧接管保存**：所有平台的发布窗口关闭前，由脚本调 creator 同款接口和完整 body 格式
3. **alert 调试链路**：保存成功/失败均弹 alert，含详细诊断字段

### 关键改动

#### 1. `domain-config.js`
- 修正 `platformLoginCookies`：
  - 搜狐 → `['sct', 'passport', 'ppinf', 'pprdig']`
  - 腾讯 → `['uin', 'p_uin', 'skey', 'p_skey']`
- 新增 `platformIdentityCookies`：账号身份字段，用于精确比对
  - 搜狐 → `['passport', 'ppinf']`
  - 腾讯 → `['uin', 'p_uin']`

#### 2. `main.js`
- 新增 `hasValidLoginCookies(windowSession, platform)` 辅助函数
- 新增 `matchAccountIdentity(windowSession, sessionData, platform)` 辅助函数
- 改造 `if (effectiveSessionData)` 流程：本地优先（命中 → 跳过 sessionData 覆盖）
- 改造 `newWindow.on('close')` close handler：脚本优先（`__publishSaveSession__`）+ 主进程兜底
- `getPublishBackendAccountId` 追加 `element.media_auth_id` 兜底
- `saveWindowSessionToBackend` 末尾对 BrowserView 注入 alert，含完整诊断

#### 3. `content-preload.js`
- `buildPublishWindowJob` 中 `backendAccountId` 兜底链路追加 `media_auth_id`

#### 4. `injected-scripts/common.js`
- 新增 `window.__publishSaveSession__(platform)` 通用保存函数
- 含 10 个平台的 `apiPath / domains / getUserInfo` 配置
- 调用 creator 同款接口和 body 格式上报
- 弹 alert 显示结果

#### 5. `injected-scripts/{tengxvnhao,souhuhao}-creator.js`
- 授权接口调用后也加 alert（用于对比授权 vs 发布保存）

#### 6. `injected-scripts/common.js`
- 改造 `showOperationBanner`：浮动卡片 + 可折叠（不挤压页面布局，对 SPA / 100vh 友好）

## 七、注意事项

### ⚠️ 小红书 / 视频号
当前 `getUserInfo` 用 DOM 选择器，在创作中心首页能取到，但发布页 DOM 结构可能不同导致取不到值，会走 publishData 兜底。**实际验证时若 alert 报错，需要在 common.js 中调整选择器或换接口获取**。

### ⚠️ Toutiao 多域名
头条涉及 `www.toutiao.com / .toutiao.com / xxbg.snssdk.com / .bytedance.com` 多个域，已在配置中覆盖。

### ⚠️ 兜底链路
脚本侧保存失败时，会自动兜底走主进程 `saveWindowSessionToBackend`（轻量 `{id, cookies}` 格式），可能仍被后台拒绝返回 400。如果兜底也失败，会有详细的诊断 alert。

### ⚠️ 时机
当前保存触发点是 `newWindow.on('close')`（窗口关闭时）。如果发布过程中崩溃或强制关闭，可能不会触发保存。
