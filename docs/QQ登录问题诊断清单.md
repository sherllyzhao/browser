# QQ 登录腾讯号问题诊断清单

## 现象
- 客户环境：360 浏览器提示"环境不安全"，运营助手无提示、登录失败
- 开发者环境：所有浏览器（Chrome、运营助手）都能正常登录

## 根因方向
**排除浏览器/代码问题** → **锁定客户机器环境问题**

---

## 诊断步骤（请客户配合）

### 1️⃣ 关闭 360 安全软件（最可疑）

**操作**：
1. 退出 360 安全卫士
2. 退出 360 杀毒
3. 重启运营助手，再试 QQ 登录

**预期**：如果 360 在拦截，关掉后应该能登录。

---

### 2️⃣ 检查系统 Hosts 文件

**位置**：`C:\Windows\System32\drivers\etc\hosts`

**检查内容**：搜索是否有 `qq.com` / `ptlogin2` / `om.qq.com` 相关劫持

```
# 不应该出现这类内容：
127.0.0.1 ptlogin2.qq.com
0.0.0.0 om.qq.com
```

**修复**：删掉相关行，保存，重启浏览器。

---

### 3️⃣ 检查网络代理/VPN

**操作**：
- 打开 `设置 → 网络和 Internet → 代理`
- 确认"自动检测设置"关闭、手动代理关闭
- 如果有 VPN，暂时断开

**原因**：企业网络/代理可能拦截 QQ 登录请求。

---

### 4️⃣ 检查浏览器 Cookie 权限

**运营助手数据目录**：
- 开发环境/安装版：`%APPDATA%\运营助手`
- 便携版：`%LOCALAPPATA%\运营助手-Portable`

**操作**：
1. 关闭运营助手
2. 删除整个数据目录
3. 重启运营助手，再试登录

**原因**：Cookie 数据库损坏或权限被系统策略锁定。

---

### 5️⃣ 对比测试（定位问题层级）

| 浏览器 | 操作 | 结果 |
|--------|------|------|
| Chrome | 访问 https://om.qq.com 并 QQ 登录 | ？ |
| Edge | 访问 https://om.qq.com 并 QQ 登录 | ？ |
| 运营助手 | QQ 扫码授权 | ？ |

**分析**：
- Chrome/Edge 也不行 → 系统级拦截（360、网络、Hosts）
- Chrome/Edge 能行，运营助手不行 → 数据目录/权限问题

---

### 6️⃣ 查看运营助手 Console 日志（技术细节）

**操作**：
1. 运营助手打开授权窗口
2. 按 `Ctrl+Shift+I` 打开 DevTools
3. 切到 Console 标签
4. 执行 QQ 登录
5. 截图所有 `[腾讯号授权]` 开头的日志

**重点看**：
- 是否有红色报错（`❌`）
- `🚀 ~ result:` 这行的返回内容
- 网络请求（Network 标签）`homeInfo` / `basicinfo` 是否返回 200

---

## 常见根因

| 症状 | 根因 | 解决方案 |
|------|------|----------|
| 360 提示"环境不安全" | 360 浏览器防护拦截 | 关闭 360 或加白名单 |
| 无提示、静默失败 | Cookie 写入被拦截 | 检查 360、杀毒软件、系统策略 |
| Chrome 也不行 | 网络/Hosts/代理 | 检查步骤 2、3 |
| 只运营助手不行 | 数据目录权限 | 删除数据目录重建（步骤 4） |

---

## 快速排查命令（请客户在 CMD 运行）

```cmd
REM 1. Ping QQ 域名（检查 DNS）
ping om.qq.com
ping ptlogin2.qq.com

REM 2. 检查 Hosts 劫持
findstr /i "qq.com" C:\Windows\System32\drivers\etc\hosts

REM 3. 检查代理设置
netsh winhttp show proxy
```
