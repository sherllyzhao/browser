# 发版说明模板

> 适用范围：运营助手浏览器（登录、Cookie、会话与多账号相关改动）

## 基本信息

- 版本号：`vX.Y.Z`
- 发版日期：`YYYY-MM-DD`
- 发布类型：`功能新增 / 优化 / 修复 / 紧急修复`
- 影响范围：`全部用户 / 灰度用户 / 指定环境`

## 本次更新摘要

- 新增：`一句话总结新增能力`
- 优化：`一句话总结优化点`
- 修复：`一句话总结修复点`

## 详细变更

### 1. Cookie 与会话

- 保存 Cookie 时包含 `expirationDate`（秒级 Unix 时间戳）。
- 会话恢复时优先使用原 `expirationDate`；缺失时使用“当前时间 + 1 年”兜底。

### 2. 登录有效性规则

- 登录状态由三部分共同决定：
- 本地 Cookie
- `globalStorage.login_expires`
- 后端接口鉴权结果（如 `401`）
- 说明：仅修改本地 Cookie 过期时间，不能实现永久登录。

### 3. 过期提醒体验

- 已过期提示：检测到 token 已过期时弹窗提示并跳转登录页。
- 提前预警：默认在到期前 10 分钟提醒一次（每分钟检查；同一过期时间只提示一次）。
- 可配置项：`renderer.js` 中 `TOKEN_EXPIRY_REMINDER_SECONDS`。

## 用户可见变化（可复制到群公告）

```
本次版本优化了登录状态管理：
1) 增加登录过期前提醒（默认提前10分钟）；
2) 登录过期后会给出明确提示并引导重新登录；
3) 强化会话恢复一致性，减少“看似在线但实际已失效”的情况。
```

## 兼容性与风险

- 兼容性：`如无特殊说明，兼容现有账号与会话数据`
- 潜在风险：`填写可能影响（如首次启动会出现一次提醒）`
- 回滚建议：`回滚到上一个稳定版本 + 清理本地缓存（如需）`
